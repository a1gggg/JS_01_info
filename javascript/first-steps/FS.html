Підключення скриптів у HTML-документі:
1. З зовнішнього файлу на сервері нашого сайту:
<Фscript src="/path/to/script.js"><Ф/script>
2. Зберігання в середині файлу html:
<Фscript> ... <Ф/script>
3. З зовнішнього сайту:
<Фscript src="https://cdnjs.cloudflare.com/ajax/libs/lodash.js/4.17.11/lodash.js"><Ф/script>

Суворий режим "use strict":
Для активації суворого режиму в JavaScript, ви можете використовувати директиву "use strict" або 'use strict'. Це дозволяє вам вказати, що ви хочете використовувати строгий режим для вашого коду. Суворий режим допомагає уникнути деяких помилок і небажаних поведінок у JavaScript.
"use strict"; // далі код нижче працюватиме у сучасному режимі

Змінні:
Створення:
let x = 5; // змінна x має значення 5 (може бути змінена)
var x = 5; // змінна x має значення 5 (може бути змінена)
const x = 5; // константа x має значення 5 (не може бути змінена)

Виклик:
//alert(x); // виведе 5


Перезапис значення змінної:
let x; // оголошення змінної без ініціалізації

    x = 'Привіт!'; // присвоєння значення 'Привіт!
    alert(x); // виведе 'Привіт!'

    x = 'Світ!'; // значення змінено на 'Світ!' (змінна може бути зміненою, якщо вона створена за допомогою синтаксису let або var. Якщо змінна створена за допомогою const, то її значення не може бути змінено)
    alert(x); // виведе 'Світ!'

Присвоєння змінній значення іншої змінної:
let x = 10;

    x = y; // присвоєння значення у змінній x;

    alert(x); // виведе 5;

Правила назви змінної:
Змінна може мати будь-яке ім'я, яке починається з літери, знака підкреслення (_) або знака долара ($). Далі можуть бути літери, цифри, знак підкреслення (_) або знак долара ($). Імена змінних чутливі до регістру. При створенні використовується верблюжий регістр наприклад: myVariableName. Для створення const використовуються верхній регістр наприклад MY_VARIABLE_NAME, COLOR_RED і тд. Змінна не може починатись з цифри. Змінна не може мати пробіли, дефіси, крапки, коми, дужки та інші символи. Змінна не може мати зарезервовані слова JavaScript (наприклад: let, var, const, if, else, for, while тощо).

Правила придумування імен:
* Використовуйте імена, які легко прочитати, як-от userName або shoppingCart.
* Уникайте використання абревіатур або коротких імен, таких як a, b та c, окрім тих випадків, коли ви точно знаєте, що так потрібно.
* Робіть імена максимально описовими і лаконічними. Наприклад, такі імена погані: data і value. Такі імена нічого не говорять. Їх можна використовувати лише тоді, коли з контексту очевидно, на які дані або значення посилається змінна.
* Погоджуйте з вашою командою (та з самим собою), які терміни будуть використовуватися у проєкті. Якщо відвідувач сайту називається “user”, тоді ми маємо давати відповідні імена іншим пов'язаним змінним: currentUser або newUser, замість currentVisitor або newManInTown.
